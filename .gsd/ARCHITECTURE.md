## Data Flow

1. **Client Action:** User interacts with the UI (e.g., changes a sitie config).
2. **Local State/Optimistic Update:** UI updates immediately using React state or Tanstack Query.
3. **API Call:** `apiFetch` (in `src/lib/api-client.ts`) sends a request to the backend with the `X-Tenant-Domain` header.
4. **Backend Processing:** The remote backend (`https://nepdora.baliyoventures.com`) processes the request.
5. **Revalidation:** For published sites, server actions trigger cache revalidation (e.g., `src/lib/actions/publish-revalidation.ts`).
